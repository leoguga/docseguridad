En este capítulo se tratarán los aspectos relacionados con la
seguridad del software desarrollado y/o utilizado internamente, y la
prevención, detección y diagnóstico del software malicioso. También se
tratarán los aspectos relacionados con el hardware de los activos de
la organización.

Para los aspectos de desarrollo de software, se puede tomar como guía
en la sección de desarrollo las recomendaciones de las transparencias
y los informes CWE más relevantes, posiblemente integrándolos en las
revisiones periódicas del código, si se desean implantar.

Habrá que analizar los riesgos que supone el software malicioso, y en
base a ellos plantear las debidas medidas de prevención, detección y
recuperación.

\section{Gestión de soportes}

Las medidas que se van a tomar para la gestión de los soportes de almacenamiento, se redactan en este punto. Para la realización de las mismas, se siguen las recomendaciones de la norma UNE-ISO/IEC 17799, indicadas en el punto <<8.6 Utilización y seguridad de los soportes de información>>.

El objetivo principal por el que se sigue la norma indicada en el párrafo anterior es el de evitar daños a los activos e interrupciones de las actividades de la organización. Para ello se establecen los procedimientos operativos adecuados para proteger los documentos, soportes informáticos (tales como CDs, cintas, etc.), datos de entrada o salida y documentación del sistema, de daño, robo y acceso no autorizado.

\subsection{Gestión de soportes extraibles}

Los procedimientos a seguir para la gestión de los soportes informáticos desechables como cintas, discos o resultados impresos, seguiran los siguientes controles:

\begin{itemize}
  \item Se borrarán cuando no se necesiten más, los contenidos previos de todo tipo de soporte reutilizable del que se desprenda la organización
  \item Todo soporte del que se desprenda la organización debería requerir autorización y se debería guardar registro de dicho acto para guardar una pista de auditoría.
  \item Todos los soportes se almacenarán a salvo en un entorno seguro, de acuerdo con las especificaciones de los fabricantes.
\end{itemize}

\subsection{Eliminación de soportes}

Se eliminarán los soportes de forma segura y sin peligro cuando no se necesiten más. Podría filtrarse a personas externas información sensible si los soportes se eliminan sin precauciones. Para minimizar el riesgo con la eliminación segura de los soportes se establecen los siguientes procedimientos:

\begin{itemize}
  \item Los soportes que contengan información sensible se almacenarán y eliminarán de forma segura, por ejemplo, incinerándolos, triturándolos o vaciando sus datos para usarlos en otra aplicación dentro de la organización.
  \item La siguiente lista identifica qué elementos requieren una eliminación segura:
  \begin{itemize}
    \item Documentos sobre papel.
    \item Registros de voz.
    \item Papel carbón.
    \item Informes.
    \item Cintas de impresora de un solo uso.
    \item Cintas magnéticas.
    \item Discos o casetes extraibles.
    \item Soportes de almacenamiento óptico, incluidos los de distribución de software de fabricantes.
    \item Listados de programas.
    \item Datos de pruebas.
    \item Documentación de los sistemas.
  \end{itemize}
  \item Se registrará la eliminación de elementos sensibles donde sea posible para mantener una pista de auditoría.
\end{itemize}

\subsection{Procedimientos de utilización de la información}

A continuación se detallan los procedimientos de utilización y almacenamiento de la información de forma coherente con su clasificación para protegerla de su mal uso o divulgación no autorizada, y de acuerdo con su soporte:

\begin{itemize}
  \item Se etiquetarán todos los soportes para su gestión.
  \item Existirán restricciones de acceso para identificar al personal no autorizado.
  \item Se realizará el mantenimiento de un registro formal de recipientes autorizados de datos.
  \item Se asegurarán de que los datos de entrada, su proceso y la validación de la salida están completos.
  \item Se protegerán los datos que están en cola para su salida en un nivel coherente con su criticidad.
  \item Se almacenarán los soportes en un entorno acorde con las especificaciones del fabricante.
  \item Se marcarán todas las copias de los datos con la dirección de los recipientes autorizados.
  \item Se revisarán las listas de distribución y de recipientes autorizados a intervalos regulares.
\end{itemize}

\subsection{Seguridad de la documentación de sistemas}

La documentación de sistemas puede contener una variedad de información sensible. Para proteger la documentación de sistemas de accesos no autorizados se considerarán los siguientes controles y medidas:

\begin{itemize}
  \item Se almacenarán con seguridad la documentación de sistemas.
  \item La lista de acceso a la documentación de sistemas se limitará al máximo, y será autorizada por el propietario de la aplicación.
  \item La documentación de sistemas mantenida en una red pública, o suministrada vía una red pública, se debería proteger adecuadamente.
\end{itemize}
